# Commit Style Rules

Use descriptive conventional commits.

Formats:

feat:
refactor:
docs:
experiment:
compare:
notes:
fix:

Examples:

feat: add redux selectors iteration
experiment: compare zustand store splitting approaches
docs: update README progress for redux
notes: summarize context provider tradeoffs
refactor: align showcase task-board layouts
compare: document redux vs zustand ergonomics

Avoid:

- update
- changes
- fix stuff
- misc
