# Showcase Rules

Showcase projects demonstrate application, not exploration.

Requirements:

- same task-board domain across all showcase apps
- same core features
- similar UI structure
- implementation should reflect tool strengths

Example task-board features:

- list tasks
- add/edit/delete task
- filtering
- sorting
- task detail drawer
- optimistic updates where relevant

Goals:

- compare implementation styles
- compare developer ergonomics
- compare state ownership boundaries

Avoid:

- experimental code
- unfinished concepts
- multiple competing patterns
